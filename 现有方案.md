# 微信公众号图片批量去水印 — 现有方案

## 方案概述

基于**固定区域掩码 + LaMa 深度学习修复**的批量去水印方案，专门针对微信公众号图片左下角的 "ST" Logo + 版权信息水印。

## 目录结构

```
wechat_articles/
  <文章名>/
    images/           # 原始图片
    images_new/       # 去水印后的图片
    processed_images.txt  # 已处理记录
```

## 核心流程

### 1. 水印区域检测（固定掩码）

水印位于图片**左下角**，位置在所有图片中一致，无需动态检测：

- 掩码范围：**底部 12% 高度 × 左侧 40% 宽度**
- 使用 `numpy.zeros` 生成全黑掩码，将水印区域置为 255

### 2. 掩码扩张

使用椭圆形结构元素对掩码进行膨胀，覆盖水印边缘的半透明过渡区域：

| 预设 | 膨胀次数 | OpenCV 修复半径 | 适用场景 |
|------|---------|----------------|---------|
| `light` | 1 | 3 | 非常淡的水印 |
| `medium`（默认） | 2 | 5 | 一般水印 |
| `heavy` | 3 | 7 | 较深的水印 |

### 3. 图像修复（双引擎）

**主引擎：LaMa 深度学习修复**
- 使用 `simple_lama_inpainting` 库（SimpleLama）
- 延迟加载，首次调用时初始化模型
- 输入：BGR 图像 → RGB PIL Image + PIL Mask
- 输出：修复后的图像（注意：LaMa 可能轻微改变图像尺寸）

**降级引擎：OpenCV Navier-Stokes**
- 当 LaMa 不可用或调用失败时自动降级
- 使用 `cv2.inpaint(img, mask, radius, cv2.INPAINT_NS)`
- 效果较 LaMa 差，但无额外依赖

### 4. 图片 I/O（支持中文路径）

使用 `open(path, "rb")` + `np.frombuffer` + `cv2.imdecode` 读取，绕过 OpenCV 不支持中文路径的问题。

## 使用方式

```bash
# 增量处理（跳过已处理的图片）
run_remove_watermark.bat

# 强制重新处理所有图片
run_remove_watermark_reset.bat

# 命令行直接运行
python batch_remove_watermark.py -p medium        # 增量处理
python batch_remove_watermark.py -p medium -f     # 强制重处理
python batch_remove_watermark.py -p heavy         # 强力修复
python batch_remove_watermark.py -d other_dir     # 指定目录
```

## 已知问题与修复记录

### 批处理文件编码问题

**问题**：`.bat` 文件中的中文字符导致 cmd.exe 解析失败，报错 `'绘按鍗板伐鍏?echo' 不是内部或外部命令`。

**原因**：Windows cmd.exe 以系统默认编码（GBK/936）读取批处理文件，即使设置了 `chcp 65001` 和 UTF-8 BOM 也无法改变解析行为。

**解决**：批处理文件中只使用 ASCII 字符，中文输出由 Python 脚本处理。

### LaMa 静默降级

**问题**：批处理运行时 LaMa 可能抛出异常，被 `except Exception` 吞掉后自动降级到 OpenCV，修复效果差但无错误提示。

**解决**：在 `except` 中添加 `print` 日志输出，显示 LaMa 失败原因和降级信息。用户可据此排查问题。

### 输出尺寸变化

**注意**：LaMa 修复后图像尺寸可能与原始图像有 1-2 像素的差异（如 446px → 448px），这是 LaMa 模型的正常行为，不影响使用。
